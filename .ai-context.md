## Authentication & Roles

Auth via Supabase. All data scoped by `workspace_id`.

| Role | Access |
|---|---|
| `super_admin` | Platform admin (`/platform-admin`) — full system, no workspace_id |
| `admin` | Workspace admin (`/control-centre`) — `admin_scope: 'workspace'` |
| `client` | Client portal |
| `supplier` | Supplier portal |

### Auth Flow (CRITICAL — do not break)

`AuthContext.fetchUserProfile` uses a **fixed safe query** — only these 7 columns:
```
role, company_name, logo_url, is_demo, status, workspace_id, admin_scope
```
**NEVER add columns or joins to this query** — it caused silent login failure when `onboarding_status` was added (column may not exist on all DB instances). `onboarding_status` and `workspaceStatus` are loaded separately in a `useEffect` after login via `fetchProfileExtras` and `fetchWorkspaceStatus`.

`fetchUserProfile` returns `'ok' | 'missing' | 'error'`:
- `'ok'` — profile loaded, proceed
- `'missing'` — no profile row (invited user not set up) → `handleMissingProfile()`
- `'error'` — transient DB/network fail → user stays logged in, no sign-out

Context providers (wrap entire app in `App.jsx`):
- `AuthContext` — currentUser, userRole, workspaceId, workspaceStatus, onboardingStatus, isSuperAdmin, isCustomerAdmin, canAccessWorkspace, needsOAuthCompletion, oauthUser
- `AdminContext` — admin-scoped data
- `SupplierContext` — supplier-scoped data
- `ClientContext` — client-scoped data
- `ThemeContext` — light/dark theme

## Joinlist / Onboarding System (NEW — 2026-03-15)

New self-signup users are NOT immediately active. Flow:

```
Signup → Login → /onboarding (5-step wizard) → /pending-approval → super-admin approves → workspace active
```

### Workspace states
- `pending` — new self-signup, awaiting approval (default for new signups)
- `active` — approved, full access
- `archived` — rejected

### Profile onboarding_status states
- `not_started` — new self-signup, hasn't done wizard
- `completed` — wizard submitted, awaiting admin approval
- `approved` — workspace active, full access (also the default for invited users)
- `rejected` — admin rejected

### ProtectedRoute gating logic
If `userRole === 'admin' && workspaceStatus === 'pending'`:
- `onboardingStatus === 'not_started'` → redirect to `/onboarding`
- `onboardingStatus === 'completed'` → redirect to `/pending-approval`

Note: `workspaceStatus` loads asynchronously (separate useEffect) — there may be a brief pass-through before redirect fires. This is acceptable.

### DB Migration
`supabase/migrations/20260319000000_joinlist.sql`:
- `workspaces.status` default changed to `'pending'`
- `workspaces.onboarding_data JSONB` added
- `workspaces.onboarding_completed_at TIMESTAMPTZ` added
- `approve_workspace_joinlist(workspace_id, admin_id)` RPC — activates workspace, approves profiles, sends notification
- `reject_workspace_joinlist(workspace_id, admin_id)` RPC — archives workspace

**⚠️ This migration MUST be run in Supabase SQL Editor before the joinlist features work.**
